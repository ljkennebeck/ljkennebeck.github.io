Finance Tracker

Link: Github | Site
Tech: Node.js, Express, Next.js, TypeScript, PostgreSQL, Prisma ORM, Tailwind, Vercel

Overview:
A full-stack finance app that lets users track, categorize, and securely connect accounts with others for collaborative budgeting. Built with a Node.js/Express backend, PostgreSQL database, and Prisma ORM, it features secure authentication using bcrypt and JWT. The modern Next.js frontend provides a smooth, intuitive user experience, and the app is deployed on Vercel.

Personal Notes:
I wanted to build something with Express and chose an expense tracker for its practical use. The original version was made for my mom, but it ran locally in Python and only created a text file, so it wasn’t very usable. It was initially planned as a mobile app with receipt scanning, but I focused on a web version instead and dropped the scanning feature early on.

GraphLab

Link: Github
Tech: Python, Django, Pandas, PostgreSQL, JavaScript

Overview:
A Django monolithic app that lets users create simple, custom datasets, visualize them with JavaScript-powered graphs, and download their data as a CSV (via Pandas) or a static graph (via Matplotlib). Built with an MVT architecture and a PostgreSQL database, it features pagination, user authentication with a dedicated email system for password resets, and robust data validation.

Personal Notes:
This project was built with the sole intention of gaining a further understanding of web development by exploring a different framework and gaining a new perspective. It was an enjoyable experience that was made easier by the solid foundation I already had. Django was a framework that I found very confusing when it was first introduced to me at the end of an Intro to Python class, so it was rewarding to revisit.
